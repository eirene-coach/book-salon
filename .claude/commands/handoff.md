# 세션 종료 및 인수인계

현재 세션의 작업을 정리하고, 다음 세션을 위한 인수인계 기록을 남깁니다.

## 보드 찾기
- `board_list`로 내 보드 목록을 가져옵니다.
- CLAUDE.md에 `Board ID`가 있으면 해당 보드를 사용합니다.
- 없으면 `board_list` 결과에서 보드를 선택합니다 (1개면 자동 선택).
- 보드가 없으면 "Dev Board가 없습니다. Vuild 웹에서 프로젝트의 BUILD 단계를 시작하세요." 안내 후 종료.
- MCP 연결 실패 → `/setup`으로 MCP 설치 안내.

## 실행 순서

### 1. 현재 상태 수집
- `board_get_summary`로 보드 현황
- `board_list_tickets`로 in_progress, review 상태 티켓 확인
- in_progress 티켓의 `board_get_ticket`로 최근 활동 확인

### 2. 미완료 작업 정리
- 각 in_progress 티켓에 `board_add_activity`로 현재 진행 상황 기록

### 3. git 상태 확인
- 미커밋 변경사항 확인 → 있으면 커밋 여부 확인

### 4. 세션 상태 스냅샷 저장
`memory/session-state.md` 파일을 아래 형식으로 갱신합니다.
이 파일은 다음 세션 시작 시 기술적 컨텍스트를 빠르게 복원하기 위한 것입니다.

```markdown
# Session State

> 마지막 갱신: [현재 날짜시간]

## 브랜치
- 현재: [git branch 결과]
- 미커밋 변경: [있음/없음, 있으면 파일 목록]

## 실패 중인 테스트
- [있으면 실패 테스트 목록, 없으면 "없음 (bin/ci 통과)"]

## WIP (작업 중인 파일)
- [이번 세션에서 수정한 주요 파일 목록]

## 다음 작업
1. [가장 우선할 기술적 작업]
2. [그 다음]

## 블로커
- [해결 안 된 기술적 문제, 없으면 "없음"]

## [NEEDS_DECISION] 대기
- [에스컬레이션 대기 중인 항목, 없으면 "없음"]
```

**주의**: 비즈니스 상태(티켓 진행률 등)는 Dev Board에 있으므로 여기엔 기술적 정보만 기록합니다.

### 5. 인수인계 요약
```
■ 이번 세션 요약
  - 완료: [done된 티켓들]
  - 진행 중: [in_progress 티켓들 + 현재 상태]
  - 리뷰 대기: [review 티켓들]

■ 다음 세션에서 할 일
  1. [우선 처리 작업]
  2. [다음 작업]

■ 주의사항
  - [블로커나 이슈]

■ 세션 상태
  - memory/session-state.md 갱신 완료
```
